What it does

A small CLI (tab) and a /tabs skill let your session reference, read, and act on the tabs you already have open — by number, by name, or by "what I'm looking at."

🔖 Stable handles

Every open tab gets a short handle — t1, t2 — that survives refreshes. Say "read t7" or "summarize the figma tab" instead of copy-pasting a long URL. The same handle names the tab on the board and in your session.

📖 Read past the login wall

The headline. read pulls a tab's live, rendered text from your already-authenticated browser — the logged-in SharePoint doc, the Figma page, the gated newsletter — and extracts the main article, dropping nav and footer chrome. Add --md to get it back as clean Markdown — headings, lists, links, tables intact — ready to drop in as context.

🔍 Grep what's open

Search the actual on-page content of every tab, not just titles. "Which tab mentions the pricing change?" returns the handles, with excerpts. Find by what you read, not what you remember calling it.

📑 Pull a cluster in one call

Hand several tabs to one read t29 t30 t4 and the whole working set lands in context at once — the three articles, the spec, and the competitor page, ready to synthesize.

👁️ "What am I looking at?"

read active grabs the frontmost tab — the page on your screen right now — so you can ask about it without naming it. The agent sees what you see.

🗂️ A board, too

A local board clusters your tabs into the tasks they belong to, flags the ones gone stale (untouched 3+ days), and shows each tab's handle — so you remember what you started and can hand any of it to the agent.

How it works

Handle calls no AI API of its own. It reads your real Chrome locally; your Claude Code session does the thinking. One shared file of handles is all the board and the agent need.

01 · reference

Name the tab

You point at a tab the way you'd say it out loud — "the figma tab," "tab 7," "what I'm looking at." Handle resolves it to a stable handle.

02 · read

Pull the content

It runs JavaScript inside that live tab to extract the rendered text — logged in, past the wall — for a few hundred tokens. Cached by default; --live for a fresh full page; --md to get clean Markdown (headings, lists, links, tables) for use as context.

03 · act

Work with it

The text is in the conversation. Summarize it, turn it into a task, compare it across tabs, ingest it — whatever the session's skills and tools can do.

# in any Claude Code session — say it in plain English, or run tab directly
tab list                  # every open tab: handle · group · staleness
tab grep "fable pricing"     # search the on-page content of every tab
tab read active           # the tab you're looking at, right now
tab read t3                # cached & instant
tab read t29 t30 t4        # a whole cluster in one call
tab read t3 --live         # fresh full page, nav stripped
tab read t3 --md           # convert to clean Markdown — headings, lists, links, tables

Or plug it into any agent — as an MCP

Handle ships as a local MCP server, so any MCP client — Claude Code, Claude Desktop, another agent — can read your tabs as first-class tools, no skill files or paths to wire up. It runs on your machine and reads your Chrome; it never exposes anyone else's tabs.

# register the local server with Claude Code
claude mcp add handle -- node ~/claude-apps/handle/mcp/server.js

🧰 Tools, not stdout

Seventeen tools — list_tabs, find_tab, grep_tabs, ask_tabs, read_tab, screenshot_tab, save_tabs, recall_bundle, and the rest. The agent calls them directly; no shell, no parsing.

🖼️ Screenshots inline

read_tab on a Figma or PDF returns the capture as an inline image the model reads with vision — the catch-all, delivered over the protocol.

📦 Zero-dependency Node

A thin wrapper over the same Python engine. Speaks MCP over stdio, no npm install. Same family as sessions-mcp.

Use it

macOS + Google Chrome. Pure Python standard library — no dependencies, no API key. The agent side runs through your existing Claude Code session.

1

Clone it

git clone https://github.com/abhitsian/handle.git ~/claude-apps/handle
2

Let Chrome read its own pages

Enable Chrome → View → Developer → Allow JavaScript from Apple Events. This is what lets read pull a tab's live, logged-in content. First run also asks for Automation permission — allow it.

3

Reference your tabs from Claude Code

Drop the /tabs skill in ~/.claude/skills/ (included) and just talk: "what am I looking at?", "summarize the figma tab," "which tab mentions the migration?" Or call the CLI yourself with python3 tab.py …

4

Open the board (optional)

cd ~/claude-apps/handle && python3 app.py
# → http://localhost:4910
